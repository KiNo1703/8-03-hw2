# Задание 1

Master-Slave репликация

Роли узлов: Асимметричные. Один сервер (Master) принимает все операции записи и обновления, а один или несколько серверов (Slave) реплицируют эти данные и используются для чтения.

Назначение:
Масштабирование чтений: Распределение нагрузки запросов на чтение по нескольким Slave-серверам.
Повышение отказоустойчивости: Slave-сервер может быть быстро назначен новым Master в случае сбоя (процедура failover).
Резервное копирование: Создание бэкапов на Slave-сервере без нагрузки на основной Master.

Преимущества: Простота настройки и поддержки, низкий риск потери согласованности данных.

Недостаток: Master является единой точкой отказа для операций записи.

Master-Master репликация
Роли узлов: Симметричные. Все серверы являются Master-ами и могут одновременно принимать как запросы на чтение, так и на запись.

Назначение:
Отказоустойчивость записей: При падении одного из Master-серверов, запись может продолжать выполняться на других узлах.
Геораспределение: Возможность размещения узлов в разных регионах для записи данных ближе к пользователям.
Преимущество: Отсутствие единой точки отказа для операций записи.

Недостатки:
Высокая сложность: Сложная настройка и администрирование.
Риск конфликтов репликации: Высокая вероятность возникновения конфликтов, если разные мастеры одновременно изменяют одни и те же данные. Требуются специальные механизмы для их обнаружения и разрешения.
