# Задание 2

1. Отличия режимов сканирования с точки зрения сетевого трафика
Каждый режим сканирования в nmap использует разные типы TCP-флагов (или UDP-пакеты) для проверки состояния портов, что влияет на трафик:

SYN Scan (-sS)	TCP-пакет с флагом SYN	Отправляется только SYN пакет. Если порт открыт, сервер отвечает SYN/ACK, после чего сканер отправляет RST для разрыва соединения (полное TCP-соединение не устанавливается).

FIN Scan (-sF)	TCP-пакет с флагом FIN	Отправляется FIN пакет (обычно используется для завершения соединения). Закрытые порты отвечают RST, открытые — игнорируют пакет (согласно RFC 793).

Xmas Scan (-sX)	TCP-пакет с флагами FIN, PSH, URG	Пакет с несколькими установленными флагами (название от "рождественской ёлки" — все флаги "горят"). Поведение аналогично FIN-сканированию: закрытые порты отвечают RST, открытые — игнорируют.

UDP Scan (-sU)	UDP-пакет	Отправляется пустой UDP-пакет. Если порт закрыт, сервер отвечает ICMP-пакетом Port unreachable. Открытые UDP-порты обычно не отвечают, что делает сканирование медленным и ненадёжным.


2. Как отвечает сервер

Ответы сервера зависят от состояния порта и режима сканирования:

SYN Scan	SYN/ACK  сканер отправляет RST для разрыва.	RST/ACK	Нет ответа или ICMP-ошибка.

FIN Scan	Нет ответа (пакет игнорируется).	RST	Нет ответа или ICMP-ошибка.

Xmas Scan	Нет ответа.	RST	Нет ответа или ICMP-ошибка.

UDP Scan	Нет ответа (или приложение отправляет данные).	ICMP Port unreachable (тип 3, код 3).	Нет ответа
